# Capital Radar Canonical Question Map

## Purpose

This map defines the fixed analytical questions Capital Radar asks before selecting evidence, relationships, historical analogs, and portfolio translation.

The goal is not to collect indicators. The goal is to assemble the smallest useful evidence bundle that answers each question with enough accuracy, traceability, and historical context to support investor decision-making.

## Operating chain

```text
Canonical Question
-> Evidence Bundle
-> Primary Relationships
-> Relationship Ranking
-> Configuration State
-> Historical Analog
-> Portfolio Translation
```

## Design principle

```text
Asset classes are factual evidence.
Canonical questions organize the evidence.
Relationships reveal market behavior.
Historical analogs provide memory.
Portfolio translation prepares action.
```

## Relationship ranking rules

For every canonical question, relationship candidates should be scored by:

```text
Divergence
Convergence
Non-confirmation
Regime shift
Historical similarity
Capital importance
Data quality
```

## Historical quality rule

Long historical memory is valuable, but confidence must be explicit.

```text
Good evidence: direct modern series available.
Usable evidence: partial series or strong proxy available.
Partial evidence: analogically useful but data mismatch must be disclosed.
```

Pre-1960 regimes should be used carefully, not discarded.

## Visual implication

The Macro page should not only display asset classes. It should show:

```text
1. Current configuration
2. Most informative relationships
3. Selected relationship overlay chart
4. Historical analog candidates
5. Asset-class evidence library
6. Portfolio translation placeholder
```

Asset classes remain visible as ground truth, but relationships become the hero of the intelligence section.
