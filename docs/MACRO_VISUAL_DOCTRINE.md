# Macro Visual Doctrine

## Purpose

Capital Radar must communicate macro conditions through annotated data rather than explanatory prose.

The Macro surface should not become a long research note. It should become a visual evidence instrument where the dataset itself tells the story through current values, historical reference points, percentile position, regime markers, and missing-evidence disclosure.

## Core law

```text
Raw data
-> annotated data
-> comparative data
-> minimal text
```

Text is allowed only when it labels the visual evidence, names the dataset, identifies missing evidence, or states the permitted portfolio action. Text should not replace the chart.

## Section responsibility

Macro is responsible for market landscape evidence.

Holdings is responsible for portfolio translation.

Macro should show what the asset-class landscape is doing. Holdings should show what that landscape implies for existing positions, sizing, ruled zones, invalidation, and portfolio action. Do not over-load Macro with portfolio implication logic if the same decision belongs more precisely in Holdings.

## Forbidden default pattern

```text
Asset class
-> paragraph explanation
-> interpreted label
-> chart as decoration
```

This creates commentary, not intelligence.

## Required default pattern

```text
Asset class question
-> raw chart / numeric dataset
-> current value marker
-> historical set-point marker
-> percentile / regime band
-> missing evidence flag
-> optional action label
```

The chart is the primary analytic object. The interpretation is secondary.

## Visual grammar

Every asset-class workbench should prioritize:

1. Current value directly on or beside the chart.
2. Historical value markers directly on or immediately under the chart.
3. Percentile or regime position where available.
4. Comparison chips using numbers, not prose.
5. Suppressed/missing markers where data coverage does not support a claim.
6. Minimal action label only after the data is visible.

## Asset-class governing questions

Each asset class should visually answer one governing question.

| Asset class | Governing visual question |
|---|---|
| Money / Cash | What is the reward for waiting? |
| Sovereign Bonds / Duration | What is the price of time? |
| Credit | How much repayment trust exists? |
| Equity Ownership | How broadly is productive surplus being bought? |
| Long-Duration Growth / Innovation Optionality | How far into the future is capital willing to pay? |
| Real Assets | How is tangible collateral being priced against leverage and income? |
| Commodities / Inputs | How scarce is the physical world? |
| Monetary Alternatives | Is capital questioning sovereign money or chasing liquidity beta? |
| FX / Dollar | Where is global capital choosing monetary safety and funding access? |
| Volatility / Insurance | How much disorder is being priced? |

## Duration example

The Duration workbench should eventually show:

```text
PRICE OF TIME

Current:
10Y nominal yield
10Y real yield
10Y breakeven
2s10s curve
3m10y curve

Historical markers:
1994 bond shock
2000 dot-com peak
2008 GFC
2020 COVID shock
2022 inflation/rate shock
Current
```

The preferred visual is not a paragraph about Duration. It is a chart/timeline where current values and historical values are shown together.

## Annotation policy

Annotations must be data-backed.

If the chart series does not cover a historical event or regime window, the marker must be suppressed and counted as suppressed. The system must not invent values.

## Mission sequence implication

Do not pause asset-class buildout for final visual refinement yet.

However, every future Mission must comply with this doctrine so that Credit, Equity, FX, Volatility, Commodities, and other workbenches do not become prose cards.

The final Macro visual redesign should happen after the Core Macro Spine has enough data coverage to design the whole system coherently.
