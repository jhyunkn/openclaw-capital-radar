# Capital Radar Kernel

## Purpose

Capital Radar is a private investment decision-support system. Its purpose is to convert market evidence into disciplined portfolio action states.

It should reduce noise, not display more information for its own sake.

The system must answer:

1. What regime are we in?
2. What is liquidity allowing or denying?
3. Which themes are gaining or losing permission?
4. Which holdings remain valid?
5. Which opportunities are becoming asymmetric?
6. What action is permitted now?
7. What would invalidate the decision?

## Product thesis

Capital Radar is a decision cockpit, not a research archive.

The public web surface should show only what helps the user decide:

- Watch
- Probe
- Add
- Hold
- Trim
- Exit

Everything else is evidence plumbing.

## Canonical homepage architecture

The homepage has four visible sections:

1. Macro
2. Decision chart
3. Holdings
4. Opportunity

### Macro

Macro is the regime and permission layer. It absorbs:

- market regime
- liquidity condition
- rates and credit pressure
- cross-asset confirmation or contradiction
- cycle interpretation
- source confidence
- action permission
- risk invalidation

### Decision chart

Decision chart is the tactical price layer. It shows:

- SPX working map
- support / resistance / target / invalidation levels
- add / trim / defense zones
- confirmation strip
- technical contradictions
- what price behavior changes the decision

### Holdings

Holdings is the portfolio ownership layer. It shows:

- current owned positions
- thesis status
- source authority
- price-zone posture
- position-sizing implication
- invalidation rule
- action state

### Opportunity

Opportunity is the asymmetry discovery layer. It shows:

- candidates under review
- evidence quality
- promotion gate
- missing evidence
- near-miss diagnostics
- allowed action state

## Latent evidence modules

The following modules are not rejected. They are latent evidence sources.

They were disabled because they were not sufficiently compressed into the mission of the web.

### Cross-Asset Lens

Return condition:

- only if it strengthens Macro by showing confirmation, contradiction, liquidity pressure, or intermarket divergence.

Visible form:

- compact Macro subpanel or row.

Do not return as:

- standalone section.

### Asset Allocation Route

Return condition:

- only if it clarifies permission: risk-on, risk-off, barbell, cash, duration, crypto, equity, or tactical exposure logic.

Visible form:

- Macro permission note
- Holdings sizing implication
- Opportunity promotion gate

Do not return as:

- independent allocation dashboard unless approved.

### Market Tape

Return condition:

- only if it clarifies near-term confirmation, contradiction, breadth, momentum, or reversal risk.

Visible form:

- Macro confirmation strip.

Do not return as:

- news feed or ticker tape.

### Kostolany Egg

Return condition:

- only if it clarifies cycle regime and expected behavior under interest-rate/liquidity conditions.

Visible form:

- Macro cycle note or small regime badge.

Do not return as:

- standalone educational diagram.

### Trust / Data Quality

Return condition:

- always relevant, but should be distributed into the relevant section.

Visible form:

- source tier
- freshness
- confidence
- missing evidence
- proxy warning

Do not return as:

- separate trust panel.

### System Health

Return condition:

- only for edit-readiness, build status, data freshness, or internal operator confidence.

Visible form:

- internal report or small badge.

Do not return as:

- public product section unless approved.

## Analysis template

All decision research should use this structure:

1. Market regime
2. Liquidity condition
3. Sector/theme rotation
4. Ticker-specific thesis
5. Valuation/expectation gap
6. Entry timing
7. Invalidation signal
8. Position-sizing logic
9. Risk scenario
10. Final action state

## Action-state definitions

### Watch

Thesis may be interesting, but evidence, price, timing, or liquidity does not permit capital yet.

### Probe

Small starter position may be justified. Risk is real, sizing must be limited, and invalidation must be explicit.

### Add

Existing or new position has improved risk/reward, thesis is intact, and price/timing allows more exposure.

### Hold

Existing position remains valid, but new capital is not clearly justified.

### Trim

Risk/reward is weakening, position has moved beyond justified exposure, or regime pressure requires reducing size.

### Exit

Thesis is broken, invalidation triggered, or better capital use is clearly available.

## Design standard

The interface should feel restrained, institutional, precise, and decision-oriented.

Avoid:

- startup landing-page energy
- generic market commentary
- dense research archive behavior
- over-explained educational content
- decorative complexity

Prefer:

- compressed evidence
- clear hierarchy
- action language
- invalidation logic
- source confidence
- calm visual density

## Operating law

No visible element belongs on Capital Radar unless it improves decision quality.

Evidence is allowed only when it changes permission, sizing, invalidation, or action state.
